I will implement the "Daily Base Location" feature to allow users to be visible throughout the day even when offline, while prioritizing online users.

### 1. Backend Logic (Firestore & Filtering)
*   **Update `PositionDoc` Schema** in `lib/positions.ts`:
    *   Add `manualBaseExpiresAt`: Timestamp to keep the position visible for ~24h.
    *   Add `isManualBase`: Flag to identify these positions.
*   **Update `subscribeNearbyUsers`**:
    *   Modify the filter to **keep users** if they have a valid `manualBaseExpiresAt` (even if `updatedAtMs` is old).
    *   Update the sort order: **Render offline/base users first**, then online users (so online users appear "on top" on the map).
    *   Expose `lastActive` in the `NearbyUser` object so the UI can show the correct status.
*   **Add `setDailyLocation` function**:
    *   Saves the user's location with a 24-hour expiration.

### 2. UI Component (`DailyLocationPrompt`)
*   Create `components/DailyLocationPrompt.tsx`.
*   **Logic**:
    *   Checks local storage to see if the question was answered today.
    *   If not, opens a Modal asking "Où es-tu aujourd'hui ?".
*   **Options**:
    *   "📍 Ici" (Uses GPS).
    *   "🏠 Maison" (Placeholder for future feature, or just manual pick).
    *   "Ignorer".

### 3. Integration
*   Add `<DailyLocationPrompt />` to `app/(tabs)/home.tsx`.
*   The existing Home UI will automatically handle the display:
    *   **Online users**: Green dot (already implemented).
    *   **Base Location users**: Will show as "Vu à HH:mm" (since their timestamp will be fixed at the time they set it), effectively fulfilling the "visible without being online" requirement.
