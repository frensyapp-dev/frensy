# Plan de correction des problèmes bloquants pour la soumission App Store

&#x20;

## 1. Sécurité des clés API (Google Maps)

* **Action** : Supprimer toute clé Google Maps en dur dans `app.json`.
* **Action** : Créer un fichier `.env` et y stocker la clé via la variable\
  `EXPO_PUBLIC_GOOGLE_MAPS_API_KEY`.
* **Vérification** : `app.config.ts` est déjà configuré pour lire les variables d’environnement.
* **Rappel critique (indispensable)** :\
  La clé doit impérativement être **restreinte dans la Google Cloud Console** :
  * iOS : Bundle ID `com.math0.frensy`
  * Android : Package `com.math0.frensy`
  * APIs strictement nécessaires uniquement
  👉 Sans ces restrictions, la clé reste exploitable même via `.env`.

***

## 2. Permissions iOS (Info.plist)

* **Action** : Ajouter `NSMicrophoneUsageDescription` dans `app.config.ts`\
  (requis par `expo-image-picker` pour l’enregistrement vidéo).
* **Action** : Vérifier la configuration des notifications push :
  * Justification claire de l’usage
  * Déclaration correcte via `expo-notifications` si utilisé
* **Décision volontaire** :\
  ❌ **Ne pas ajouter** `NSUserTrackingUsageDescription`
  * Aucune librairie de tracking publicitaire détectée dans le projet
  * Ajouter cette permission sans l’utiliser constitue un motif de rejet (“Permission Abuse”)

***

## 3. Politique de confidentialité

* **Action** : Générer deux fichiers :
  1. `privacy.html`
     * Page HTML/CSS professionnelle
     * Prête à être hébergée (Vercel, GitHub Pages, etc.)
     * Accessible publiquement sans authentification
  2. `PRIVACY.md`
     * Contenu brut servant de référence interne / version Markdown
* **Contenu requis** :
  * Données collectées
  * Géolocalisation
  * Firebase / stockage des données
  * Achats intégrés
  * Durée de conservation
  * Droits utilisateur (suppression, accès)
  * Contact
  * Conformité RGPD
* **Action** : Mettre à jour `app.config.ts` avec le champ `privacyPolicyUrl`\
  (URL placeholder à remplacer une fois la page hébergée).

***

## 4. Vérification des fonctionnalités sociales obligatoires

* **Constat** : Les fonctionnalités suivantes sont bien implémentées dans le code :
  * Signalement
  * Blocage
  * Suppression de compte
  * Vérification d’âge (18+)\
    (`lib/report.ts`, `lib/block.ts`, etc.)
* **Vérification UI** :
  * Les boutons **Signaler** et **Bloquer** sont présents et fonctionnels dans\
    `app/user/[id].tsx`
  * Ces actions doivent être clairement visibles et accessibles depuis :
    * Profil utilisateur
    * Chat
    * Paramètres si applicable

***

## 5. Classification d’âge (App Store Connect)

* **Constat** : Le code empêche l’inscription des utilisateurs de moins de 18 ans.
* **Action lors de la soumission** :
  * Sélectionner **17+ (Mature)** sur App Store Connect
  * Répondre **Oui** aux questions concernant :
    * Rencontres
    * Contenu généré par les utilisateurs
    * Interactions sociales
